First, there are native MMD-like softwares (for animation/rendering, no modeling capabilities):

- [MikuMikuMoving](https://sites.google.com/site/mikumikumovingeng/)
- [NexGiMa](https://www.nekosmb.com/nexgima/)
- [MikuMikuDayo](https://github.com/pennennennennennenem/MikuMikuDayo)
- [nanoem](https://github.com/hkrn/nanoem)

And there are MMD-compatible softwares based on web technologies ([babylon-mmd](https://github.com/noname0310/babylon-mmd), three.js, etc.):

- [MMD modoki](https://github.com/togechiyo/MMD_modoki)
- [Reze Studio](https://github.com/AmyangXYZ/reze-studio)
- [AnimaStage](https://github.com/gtausa197-svg/AnimaStage-Pro)

Alternatively, you can import PMX via addons in non-MMD softwares. Some models prohibit this use case, so you should be careful not violate terms of use. There are many softwares that can import MMD models.

- Blender with [MMD Tools](https://extensions.blender.org/add-ons/mmd-tools/)
- Unity with [MMD4Mecanim](https://stereoarts.jp)
- UE5 with [IM-for-UE5](https://github.com/axilesoft/IM-for-UE5)
- Godot with [godot-pmx](https://github.com/V-Sekai/godot-pmx) or [Godot-MMD](https://godotengine.org/asset-library/asset/3303)
- Maya with [mmdpaimaya](https://github.com/phyblas/mmdpaimaya) or [Maya MMD Tools](https://github.com/yohawing/maya_mmd_tools)
- Cinema 4D with [C4D MMD Tool](https://github.com/AiMiDi/C4D_MMD_Tool)
